¶Ajax

  • 即异步的JavaScript和XML

    • 与服务器进行数据交互
    • 异步交互：可以在不重新加载整个页面的情况下，与服务器交换数据并更新部分网页的技术

  • 同步异步

  • 原生Ajax

    • 客户端的Ajax请求
      • 1.在VS Code中创建Ajax文件夹
      • 2.创建XMLHttpRequest对象，用于和服务器交换数据，也是原生Ajax请求的核心对象
      • 3.调用对象的open()方法设置请求的参数信息，例如请求地址，请求方式
      • 4.通过绑定事件的方式，来获取服务器响应的数据
      • 5.通过浏览器打开页面，请求点击按钮，发送Ajax请求，显示结果

¶Axios

  • 是简单的发送Ajax请求的技术
  • 对原生的AJAX进行封装，简化书写
  • Axios官网：https://www.axios-http.cn

¶Axios基本使用 1.3.1

  • 1.引入Axios文件

  • 2.使用Axios发送请求，并获取响应结果

    • 官方提供很多api
    • 其中两种：
      • get请求 将请求参数拼接到路径的后面
      • post请求

  • axios()是用来发送异步请求的，小括号中使用 js的JSON对象传递请求相关的参数

¶请求方法的别名

方法描述
axios.get(url [, config])发送get请求
axios.delete(url [, config])发送delete请求
axios.post(url [, data[, config]])发送post请求
axios.put(url [, data[, config]])发送put请求

//通过axios发送异步请求-get

   }).then(result => {
       console.log(result.data);

axios.get("http://yapi.smart-xwork.cn/mock/169327/emp/list").then(result => {
    console.log(result.data);
})

¶前后台分离开发

¶前后台分离开发介绍

  • 接口文档：后台开发者根据产品经理提供的产品原型和需求文档所撰写的
  • 流程：【需求分析】→【接口定义（API接口文档）】→【前后端并行开发（遵守规范】→【测试（前端、后端）】→前后端联调测试
    1. 需求分析：阅读需求文档，分析、理解需求
    2. 接口定义：查询接口文档中关于需求的接口的定义，包括地址，参数，响应数据类型等等
    3. 前后台并行开发：各自按照接口文档进行开发，实现需求
    4. 测试：前后台开发完毕，各自按照接口文档进行测试
    5. 前后段联调测试：前端工程请求后端工程，测试功能

¶YAPI

¶YAPI介绍
  • 官网地址：YApi Pro-高效、易用、功能强大的可视化接口管理平台
  • 主要提供2个功能：
    • API接口管理：根据需求撰写接口，包括接口的地址，参数，响应等等信息
    • Mock服务：模拟真实接口，生成接口的模拟测试数据，用于前端的测试
¶接口文档管理
  • 个人空间，选择项目列表→添加测试项目
  • 点击进入创建的项目，先添加接口的分类
  • 选择当前创建的分类，创建接口信息
  • 来到接口的编辑界面，对接口做生层次的定制。例如：接口的参数，接口的返回值等
  • 添加接口的请求参数
  • 来到返回数据设置，选择导入json，编写和添加json格式的返回值
  • 保存上述设置后，来到接口预览界面，查询接口的信息
  • 最后，还可以设置接口的mock信息
  • 完成后来到接口的预览界面，点击Mock地址，发现浏览器可以直接打开并返回前面设置的期望数据

¶前端工程化

¶前端工程化的特点

  • 模块化：将js和css等，做成一个个可复用模块

  • 组件化：我们将UI组件，css样式，js行为封装成一个个的组件，便于管理

  • 规范化：我们提供一套标准的规范的目录接口和编码规范，所有开发人员遵循这套规范

  • 自动化：项目的构建，测试，部署全部都是自动完成

  • 即：在企业级的前端项目开发中，把前端开发所需要的工具、技术、流程、经验进行规范化和标准化

  • ↓↓↓vue官方提供的脚手架→完成前端的工程化

¶Table表格

¶Table组件属性详解
  • ElementUI将数据模型绑定到视图：
    • data: 主要定义table组件的数据模型
    • prop: 定义列的数据应该绑定data中定义的具体的数据模型
    • label: 定义列的标题
    • width: 定义列的宽度
  • Element组件的所有属性都可以在组件页面最下方找到
¶Pagination分页
  • 分页组件，主要通过提供分页工具条相关功能
¶Pagination组件属性详解
  • background: 添加背景颜色
  • layout: 分页工具条的布局，其具体值包含sizes, prev, pager, next, jumper, ->, total, slot 这些值
  • total: 数据的总数量
¶Pagination组件事件详解

  • size-change ： pageSize 改变时会触发

  • current-change ：currentPage 改变时会触发

  • 注意：在script代码复制时，methods属性和data同级

¶Vue路由

¶路由介绍

  • 点击页面不同功能界面，浏览器的地址栏会发生变化，路由自动更新显示与url所对应的vue组件
  • Vue官方提供了路由插件Vue Router，主要组成：
    • VueRouter：路由器类，根据路由请求在路由视图中动态渲染选中的组件
    • router-link：请求链接组件，浏览器会解析成a
    • router-view：动态视图组件，用来渲染展示与路由路径对应的组件
路由链接组件router-link员工管理
/user : UserView.vue←路由请求——
/emp : EmpView.vue路由器VueRouter
/order : OrderView.vue路由表
——更新→
路由展示组件router-view
  • VueRouter根据配置的url的hash片段和路由的组件关系去维护一张路由表
  • 页面提供一个router-link组件，用户点击，发出路由请求
  • 接着VueRouter根据路由请求，在路由表中找到对应的vue组件
  • 最后VueRouter会切换router-view中的组件，从而进行视图的更新
