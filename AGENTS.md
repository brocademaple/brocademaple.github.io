# AGENTS.md — Brocade Maple 设计准则

这份文档写给在这个仓库里干活的 AI coding agent（Codex 或别的）。目的很直接：这个页面的设计经过了很多轮筛选才定下来，每一条都不是随手选的，是某一版被否决之后留下来的。改代码之前先看这份文档，改完之后也回来对一下，别让"局部看起来合理"的小改动，悄悄把整体拉回到一个通用模板的样子。

如果一个改动会动到下面列的任何一条，**先停下来问，不要自己决定**。

---

## 这个页面是什么

一个人的项目档案馆主页，不是简历，不是作品集模板。气质是"克制的优雅" + "地中海正午的金粉风暴"，不是赛博朋克夜店，也不是常见的紫色渐变 AI 创业公司风格。

---

## 硬性约束（改之前先确认）

### 配色 —— 只有一套主题
- 海蓝金，地中海正午调：深蓝背景（`#04243f` → `#0a4f7a`），饱和的天青蓝（`#1f8fc4`），暖金（`#e8b54a` → `#ffe29a`）。
- **不要**把蓝色调暗、调灰、调成"安全的"深色模式导航蓝。这个蓝故意选得很饱和、很有阳光感，调暗了金粉风暴会失去"纸醉金迷"的浓度。
- 之前出现过的紫金绿配色已经整体删掉了，不要因为"多一个选项更灵活"就加回多主题切换。

### 字体反差是整个页面的视觉签名
- 英文用 `Cormorant Garamond` / `Playfair Display` 的斜体花体；中文统一用 `Noto Serif SC`（衬线），**不是**无衬线体。
- 这两者的反差——冷感几何衬线 vs 暖感手写花体——是整个设计唯一的"个性来源"。换成任何系统默认字体、任何无衬线中文字体，这个反差就没了，页面会变成随便一个模板。
- 字体目前从 Google Fonts CDN 在线加载，本地化方案见下方"已知待办"，但**字体本身的选择不要换**。

### 粒子风暴不是装饰性的随机飘浮
- 金粉沿一个**倾斜的扁椭圆环**顺时针匀速游动，参数：`LOOP_CENTER = {x:0.55, y:0.38}`，`LOOP_RADIUS = {x:0.52, y:0.30}`，`LOOP_ROTATION = -32°`（左下到右上的对角线方向）。
- 粒子形态是**带棱角的菱形碎片（chip）+ 闪烁星点（glint）+ 尘点（dust）**，绝对不能是圆形/气泡。这是被明确否决过的方向——早期版本用过圆形粒子，被直接打回。
- 纯 Canvas 2D 实现，没有用 Three.js / Anime.js。如果要换成更复杂的库，先确认这是必要的，不要因为"看起来更专业"就引入新依赖。

### 卡片不是纯白卡片，也不是深色玻璃卡片
- 项目卡是"金箔"质感：暖色奶油渐变（`#f8edd2` → `#ecd9a3`）+ 金色细边框 + 金色调阴影发光。这是从"冷淡的扁平奶油色"改过来的，目的是让卡片跟饱和的背景"同属一个世界"，不是贴上去的便利贴。
- 不要因为"对比度更高更易读"就换成纯白背景卡片，也不要因为"跟背景更融合"就换成深色半透明玻璃卡片——之前两种都试过类似的方向，都不是最终选择。

### 诚实性——这条最容易被悄悄破坏
- 任何看起来像"实时数据"的 UI 元素，必须是真实数据，或者明确标注是某个时间点的快照（参考"截至本稿 · 13 个仓库"这种写法）。**不要**加一个看起来会自动更新、但其实是写死的"最新动态"组件——这个东西出现过又被删掉过。
- 没有真实预览链接的项目，`demo` 字段留 `null`，弹窗显示"预览链接待补充"。**不要**为了好看伪造一个假链接，也不要因为"看起来更完整"就删掉这个诚实的占位提示。
- 封面图占位必须明确标注"占位"（虚线框 + 文字说明），不能做成看起来像真实截图的样子。
- 几个长得不一样但点击行为完全一样的链接/按钮是不被允许的（之前 hero 区三个词全部指向同一个锚点，被指出来是"装样子"，后来改成各自指向不同的真实项目/页面）。如果要加新的可点击元素，每个目的地必须是真实且不同的。

### 文案 —— 不写自我介绍
- Hero 区故意没有任何一句"我是谁/我做什么"的文案。这是用户明确要求删掉的（"let the projects speak for me"）。**不要**为了"信息更完整"重新加一段自我介绍式的文字。
- 所有文案避免设计说明腔（不写"用了海蓝雾营造氛围"这种话）和空洞的诗意自夸。克制、具体、有判断。

### 数据结构——单一数据源
- 5 个项目的信息全部在 `PROJECTS` 数组里，卡片网格和详情弹窗都从这一份数据渲染。**不要**为了方便改某个卡片的文案，就直接改 HTML 模板字符串里的硬编码文字——所有内容改动都应该改 `PROJECTS` 数组本身，否则卡片和弹窗的内容会逐渐对不上。

### 动效——克制，张力只花在一处
- 整个页面的"视觉用力点"是粒子风暴 + 标题渐变字。其他元素（卡片 hover、按钮）的动效应该是单一、克制的反馈（比如阴影加深 + 箭头旋转，这两个加起来算"一个"反馈，不要再叠加位移、缩放、径向光晕等更多动效）。
- 之前有一版卡片 hover 同时做了"上移 + 阴影 + 径向光晕 + 箭头旋转"四件事，被指出"到处都在用力，没有重点"，后来精简掉了。新加动效前先问：这个页面已经有几个动效在抢注意力了？

## 已知待办（这些可以按需推进，不算"破坏设计"）

1. **字体本地化**：现在从 Google Fonts CDN 在线加载，文件顶部有详细的 HANDOFF NOTE 注释列了每个字体需要的字重。需要离线/国内访问稳定时，换成自托管 `@font-face`。`Noto Serif SC` 字符集很大，建议子集化。
2. **项目封面图**：`PROJECTS` 里每个项目的 `image` 字段都是 `null`，等有真实截图后填入图片路径，并修改 `openModal()` 里 `elImage.innerHTML` 那一段的渲染逻辑（现在是无条件渲染占位文字，需要改成判断 `image` 是否存在）。
3. **预览链接**：`demo` 字段全部是 `null`，有真实地址后直接填，弹窗会自动显示可点击按钮，不用改渲染逻辑。
4. **GitHub 仓库数据**："13 个仓库"是手动快照，要做成实时数据需要接 GitHub API。

## 改动前的自检清单

加任何新东西之前，问自己：

1. 这个改动会不会让蓝色变暗、变灰、变"安全"？
2. 这个改动会不会让中文变成无衬线体，或者让英文花体消失？
3. 这个改动会不会引入圆形/气泡状的装饰元素？
4. 这个改动会不会让某个 UI 元素看起来比它实际做的事情更多（虚假的实时感、虚假的链接差异化）？
5. 这个改动会不会在已经有动效的地方再叠加一层新动效？
6. 这个改动有没有同时改了 `PROJECTS` 数据和某处硬编码的重复内容，导致两者不一致？

六个问题里有任何一个的答案是"会"，先停下来确认，不要直接改。
